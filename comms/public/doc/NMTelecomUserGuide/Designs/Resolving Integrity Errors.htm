Resolving integrity errors

To resolve integrity errors:

  1. Carry out a design check. The integrity errors will be listed in the Details tab.

  2. Select the integrity error from the Details tab.

  3. The Description field will provide more detail about the integrity error.

Attribute consistency

Two attributes on the same object may conflict e.g. it may not be valid for a planned object to have an installation date. For example, a mandatory field is not set.

Error Description

not set

 A mandatory field is not set.

A reference field may point to an object that no longer exists.

Derived Attribute Consistency

For performance reasons, it is sometimes necessary to copy values onto 'child' object. The stored value can potentially become inconsistent with the values it was copied from.
