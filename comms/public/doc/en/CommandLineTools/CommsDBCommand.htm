The comms_db command

The comms_db command provides a number of comms-specific database operations. It can be used in combination with the IQGeo platform myw_db command to manage your database from the command line or within a script.

The format of the command is:

comms_db <db_name> <operation> ...

The following operations are supported:

Operation Description
install Install a template model.
validate Check configuration or data.
list Show contents of a data package.
import Import a data package.

export

Dump data as a data package.
manage_tasks

Manage different aspects of tasks.

The operation comprises a set of sub-operations for specific actions:
 
list List tasks, their status and various details.
 
cancel Cancel a task with a given <task number>.
 
delete Delete all tasks that are not running or waiting for execution.
 
run Run a task with a given <task number>.

To see the parameters and flags for a given operation, use:

comms_db <db_name> <operation> --help

 

If you use the command frequently you may want to add the tools directory to your command line environment path variable.
