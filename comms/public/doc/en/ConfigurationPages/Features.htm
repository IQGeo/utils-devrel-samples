Features Configuration page

The Features Configuration page contains an extensive list of the various features potentially available for creating a network or displaying it on the map.

To configure a feature, click on the corresponding row in the table and a page appears with detailed data about that feature.

It is not the intention to discuss all possible features and fields here, but the following are worth highlighting:

Navigating to the Stored Fields tab

  1. Click the Features icon on the Configuration main page.

  2. Click the required feature in the Feature Type column. The feature details then open at the Basic tab.

  3. Select the Stored Fields tab on the Feature Details page.

Equipment reference field editor

A field editor class myw.CommsEquipRefFieldEditor is available to allow users to select the equipment for a reference field that relates equipment to the feature.

A situation where this might be used is, if your model has an address feature, and you wanted to have a reference field that relates the address to the equipment that serves it.

Figure: Main Network Manager Telecom application - Address - Details pane - Serving Equipment data item

design feature - user_group

The table on the Stored Fields tab on the Feature Details page of the design feature contains the user_group field. This field is used in the Designs section on the Comms tab of the Settings Configuration page to allow user group information to be stored with the designs.

In the list of available fields you can see the user_group entry as shown below.

Figure: Configuration > Features - design - user_group field

iqgapp_task_queue feature

The iqgapp_task_queue feature supports running Path Finder requests asynchronously. The records for this feature can be managed using the comms_db manage_tasks command. Refer to the article The comms_db command

mywcom_change_detail feature

This feature has been made non-editable from the UI and can no longer be queried.
