Server-side Triggers

For reasons of efficiency, Network Manager performs route splitting server-side. This means that client-side triggers will not fire when a route is split by inserting a structure. Client-side triggers also do not fire when changes are made by an external process via the REST API. To overcome this, Network Manager provides a limited server-side trigger mechanism that fires on changes to network data.

To use the trigger mechanism, create a trigger handler in Python and register on the NetworkView class. Trigger events are specific to the Network Manager category of the object being modified. You can register multiple handlers for the same event.

A typical use of server-side triggers is to perform auto-naming of objects as they are inserted. See Common Tasks for an example.

The Network Manager native code supports an equivalent trigger mechanism for use in IQGeo Anywhere. For more information, contact IQGeo Support.
