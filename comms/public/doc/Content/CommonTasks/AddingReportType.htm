Adding a Report Type

Network Manager includes a framework for adding new report types. To add a report type:

  1. Create a JavaScript class to gather the data and generate the report. This should normally be a subclass of Report (see below).

  2. In your application's JavaScript definition, register the report class on the ReportManager plugin. This manager handles the display and downloading of reports. It also allows you to override or remove existing report types.

  3. If necessary, add a button in the GUI so that your user can launch the report. The button should call ReportManager.preview() with an instance of your report class.

    Note: Feature and feature set reports registered with ReportManager will appear automatically in the GUI. They do not require you to add a separate button.

A report class must provide the following methods:

Type Method Purpose
static canBuildFor(app,data) Returns true if report can be generated for 'data' (a feature or feature set). Used by ReportManager to determine which report choices to display in the GUI.
  initialize(app,data) Create report instance. Should not do any major work.
async build() Gather all data required to generate report (and store it on object).
  generate(strm) Output report to 'strm' (a ReportStream). Typically involves writing header lines and one or more ReportTable objects to the stream.

See equipmentReport.js in comms/public/js/reporting for an example for a simple feature report.
