# Dev Database for IQGeo Network Manager Telecom

## Introduction

This data is primarily used for developer coding and automated testing. It can also be used as a sample database for demonstration and training. The data is a small sample of everything and doesn't attempt to scale up to actual customer networks. This is because the database needs to be small to run with automated tests where we backup and restore the database with each test.

Some samples of code are also included.

## Data

### Master data

You can get a good feel for the data in the system by going to Woodhead Hub and tracing out from it.

Structures and routes are digitized for two service areas - WH and SP. Cables are in both areas. Backbone (BB) cables are bi-directional. All other cables are uni-directional. Cabinets near Woodhead Hub have photos. All objects have unique names generated by a name manager.

-   Fiber PON network. CO is Woodhead Hub which contains racks with OLTs. These are connected down to customers ONT on Milton Road through splitters on poles.
-   Fiber backbone network. From Woodhead Hub out to Science Park and around are a number of backbone fiber cables.
-   Copper network. Starting from the copper cable 'WH-CC-001' is a small copper network extending to the east of Woodhead. This network also includes line of count assignment. The cabinet 'WH-C-05' includes a DSLAM demonstrating equipment with mixed technology.
-   RF network. The manhole 'WH-M-266' includes three optical nodes. Downstream of WH-ON-001 is a small RF network.

## Code

The following tools and functionality is provided in the module.

### Server Python + Native JS

-   Name manager - DevDbNameManager
-   Delta filter - DevDeltaFilter
-   Path finder dev tool - DevDbPathFinder

### Client JS

-   Customer connection tool
    -   Select a pole to set splitter
    -   Select an address to set customer
    -   Click Create to build
-   Virtual walkout tool
    -   Select connected linear objects e.g. do a shortest path trace
-   Feature models - mainly subclassed to make name field readonly has this is auto-generated. DevDbRoute includes method for calculated field length.
-   Design rules
    -   Conduit capacity
    -   Pole spacing
    -   Specs set
-   Schematics. These are commented out in the main JS file.
    -   Route trace schematic
    -   Cable trace schematic
-   Dev Tools
    -   Action plugin: Allows pre-canned action to be performed after application loads (to assist testing)
    -   Test report
